# Product Requirements Document (PRD)

## Product Summary

GB Studio is a desktop application for building retro adventure games targeting Game Boy-compatible workflows.

## Goals

- Enable non-programmers to create playable games end-to-end.
- Provide fast editor feedback with preview, asset editing, and scripting tools.
- Produce build outputs for ROM/web/pocket pipelines and project export flows.

## Non-Goals

- Real-time 3D authoring.
- AAA-scale collaborative cloud editing.
- Runtime modding framework beyond current project/plugin architecture.

## Users

- Solo creators and hobbyists.
- Educators teaching game design basics.
- Retro-focused developers building production releases.

## Key Features

- Scene/actor/event editor.
- Visual scripting pipeline.
- Asset pipeline for sprites, tilesets, backgrounds, audio.
- Music/sound tooling.
- Build/export flows and project templates.

## Success Metrics

- Time to first playable prototype.
- Crash-free editing sessions.
- Build success rate for starter templates.
- Repeat usage across projects.

## Risks

- Cross-platform build toolchain drift.
- Asset import edge cases.
- Performance regression in large projects.

## Milestones

- M1: Stable editor startup and project creation.
- M2: Reliable authoring workflows and undo/redo stability.
- M3: Deterministic build output and release packaging quality.
